# CycleBubble 项目完整记录

> 本文档记录了 CycleBubble 从最初的"呼吸泡泡动画修复"到最终定位为"Emotional OS（情绪操作系统）"的完整思考与修正过程。

---

## 一、产品最终定位

### 核心定义

CycleBubble 不是一个 AI 日记，不是情绪记录器，不是 AI 聊天，不是心情管理工具。

它是：

> **一个持续理解用户、记录成长轨迹、帮助人与自己、人与他人建立真实连接的 Emotional OS（情绪操作系统）。**

### 价值主张

- 帮助用户留下情绪，而不是留下数据
- 让 AI 长期理解一个人，而不是回复一次聊天
- 每一次表达，都会成为未来理解自己的线索
- AI 不只是回答问题，而是陪伴用户完成长期成长

### 与竞品的核心差异

CareBridge（同类竞品）是**数据驱动、功能导向**的健康管理工具——健康档案、周期追踪、风险评估、家庭协同、报告导出。

CycleBubble 是**情绪驱动、体验导向**的陪伴产品——情绪记录、Bubble 成长、长期记忆、匿名共鸣。

一句话区分：**CareBridge 帮你管理身体的数据；CycleBubble 帮你安放身体的情绪。前者解决"知不知道"，后者解决"被不被理解"。**

## 二、产品飞轮（最终版）

整个产品围绕以下循环设计：

```
表达（Expression）
    ↓
沉淀（Memory）
    ↓
理解（Understanding）
    ↓
连接（Connection）
    ↓
影响（Impact）
    ↓
新的表达（Expression）
```

**成长（Growth）贯穿整个循环，但不是最终页面，也不是唯一目标。** 成长是长期使用后的自然结果。

## 三、思考修正过程（关键节点）

### 阶段 1：修复泡泡动画

最初的问题：泡泡里的水波不动。

- 尝试 CSS transform translateX → 失败，周期不匹配
- 最终方案：用 `requestAnimationFrame` 直接更新 SVG path 的 `d` 属性
- 三层波浪（back/mid/front），不同振幅、频率、速度、相位

### 阶段 2：V1 — 产品结构搭建

用户提出第一个产品升级指令：

- 首页：周期状态 + Bubble + 温柔解释 + 线索卡 + 按钮
- AI 理解页（insight）：在记录和共鸣之间
- 共鸣页：匹配说明
- 医学准确性：用"这轮周期第 21 天"而非"月经期第 21 天"

这一阶段建立了基础的四页结构。

### 阶段 3：V2 — Bubble 作为绝对视觉中心

用户指令：Bubble 是活的生命体，不是装饰。

- 首页极简：只有周期状态 + Bubble + 按钮
- Bubble 成长系统：4 个维度（液体高度、流动、透明度、光泽）
- 流程：记录 → AI 理解 + 周期解释 → 共鸣（情感高潮）
- Bubble 必须在所有阶段都美观

### 阶段 4：V3 — Bubble 交互绑定

用户指令：Bubble 和用户行为要绑定。

- 共鸣互动会给 Bubble 加光点（connection = 白色，warmth = 暖色）
- AI 理解页 4 步：事件 → 触发 → 身体节奏 → 陪伴
- Bubble morph 动画（CSS transitions）
- 漂浮粒子系统
- 按钮文字："把今天留在这里"

### 阶段 5：V4 — Bubble Constitution（世界规则）

用户指令：建立完整的 Bubble 世界规则。

8 条 Bubble 宪法：
1. AI 永远隐藏在 Bubble 身后，Bubble 才是主角
2. Bubble 不展示成长算法，成长是自然结果
3. Bubble 不会评价、打分、下结论
4. 产品语言不出现 AI 相关术语（分析→理解，检测→发现，模型→陪伴）
5. 沉淀页取代"AI 正在分析"
6. DNA 驱动 Bubble 状态（亮度、饱和度、不透明度、纹理层数、粒子密度）
7. localStorage 持久化
8. 按钮文字"把今天留在这里"

Bubble DNA 模型：
- stability（稳定）：记录积累
- depth（深度）：理解的深度
- openness（开放）：与他人的连接
- vitality（活力）：生命力
- memoryLayers：记忆层
- evolution：进化阶段（reflect → remember → resonate）

### 阶段 6：V4 重构 — Emotional OS

用户提出战略升级指令：

> 重新组织成一条更有力量的故事线：记录 → AI理解 → 长期记忆 → 发现自己的变化 → 与世界产生真实共鸣。

核心变化：
- 新增**成长页**：记忆时间线 + 成长故事
- 理解页新增**记忆连接**："这是你第三次提到对「被认可」的在意"
- 首页新增**成长旁白**：随 DNA 进化变化
- Tab 从"今日/记录/共鸣"改为"今日/成长/共鸣"
- DNA 初始值预设为非零（让用户一打开就感觉"已在变化中"）

用户曾担心"5 步太长"，最终方案：把"长期记忆"和"变化发现"融进首页 Bubble 本身（DNA 成长可视化），不做成独立页面，而是用成长旁白表达。

### 阶段 7：V5 — 体验重构（最终版）

用户提出产品二次升级指令（最重要的修正）：

#### 修正 1：降低 AI 判断感

**禁止**结论式表达：
- "你就是……"
- "你的性格是……"
- "你属于……"
- "你其实一直……"

**改为**观察式表达：
- "Bubble 最近注意到一个有趣的现象……"
- "最近几次记录里，有一个重复出现的变化……"
- "你自己会不会也有这种感觉？"
- "这是最近才开始出现的新变化。"

理解页标题从"你并不是太敏感，而是一直在默默消化别人随口说出的话"改为"Bubble 最近注意到一个有趣的现象"。

#### 修正 2：成长页重新定位

成长页不应成为每天查看的页面，而是**长期惊喜**。

- 标题从"你已经不是三个月前的自己了"改为"Bubble 想和你分享一些最近才发现的变化"
- 副标题："这些变化不是结论，只是一种观察。你自己觉得呢？"
- 成长故事全部改成观察式，每条结尾留开放空间

#### 修正 3：建立三条长期理解体系

1. **Self（自己）**：用户记录的内容、情绪表达
2. **Relationship（关系）**：用户如何回应别人（安慰、分享、鼓励、陪伴）
3. **Community（社区）**：用户收藏什么、停留在哪些故事、与哪些经历产生共鸣

成长结论必须综合三个维度，不能因为一次行为直接下结论。

#### 修正 4：共鸣升级为"回应（Response）"

禁止传统评论区，避免变成社交媒体。

设计有限表达体系：
- "我也经历过"
- "谢谢你的分享"
- "抱抱你"
- "分享我的经历"（可展开输入框）

重点：温暖、克制、低社交压力。鼓励分享真实经历，不鼓励争论和闲聊。

#### 修正 5：新增影响机制

每个 Bubble 不只是属于自己的，当它帮助别人时可以留下价值。

- 成长页新增"你的陪伴"区块
- "你的经历，陪伴了 3 位正在经历相似感受的人。"
- "有人因为你的回应，收藏了一段话。"
- 陪同回应数量动态增长

**禁止**：点赞、粉丝、排行榜、热度。**强调**：影响、陪伴、连接。

#### 修正 6：产品飞轮定义

```
表达 → 沉淀 → 理解 → 连接 → 影响 → 新的表达
```

成长贯穿整个循环，但不是最终页面，也不是唯一目标。

#### 修正 7：AI 定位升级

AI 不是聊天机器人，而是：
- 长期观察者
- 成长陪伴者
- 人生镜子
- 未来自己

AI 应该越来越懂用户，而不是越来越会聊天。

## 四、产品语言规范

### 禁止使用的词

- 分析、检测、模型（AI 相关术语）
- 评分、指数、雷达图、百分比（数据平台化）
- MBTI、人格标签、性格分析
- 点赞、粉丝、排行榜、热度（社交媒体化）
- "你就是……""你的性格是……"（结论式表达）

### 应该使用的词

- 陪伴、理解、沉淀、回应
- 发现、观察、也许、好像（留有开放空间）
- 线索、变化、节奏
- "你自己觉得呢？""会不会也有这种感觉？"

### Bubble DNA 模型

```javascript
var bubbleDNA = {
  stability: 35,    // 稳定：记录积累
  depth: 28,        // 深度：理解深度
  openness: 18,     // 开放：与他人连接
  vitality: 22,     // 活力：生命力
  memoryLayers: [],     // 用户真实记录
  responseLayers: [],   // 用户回应记录（Relationship 维度）
  evolution: "remember",
  totalRecords: 6,
  totalResponses: 0
};
```

DNA 驱动 Bubble 状态：
- breatheDuration：stability 越高，呼吸越慢（越稳定）
- brightness：vitality 越高，越亮
- saturation：depth 越高，色彩越饱和
- opacity：openness 越高，液体越透明
- textureLayers：depth 越高，纹理层越多
- particleDensity：vitality 越高，粒子越多

### 成长旁白系统

DNA 总分 ≥ 240："Bubble 好像越来越懂你了"
DNA 总分 ≥ 120："Bubble 开始记住你的节奏了"
DNA 总分 < 120："Bubble 还在慢慢认识你"

### 预置数据（模拟三个月成长轨迹）

5 条种子记忆（三个月前→两周前），展示从"自我怀疑"到"觉察"的成长轨迹。

4 条成长故事（全部观察式）：
- 表达方式变化
- 恢复能力变化
- 与他人的连接变化
- 周期觉察变化

### 回应系统

4 种有限回应：
- empathy（我也经历过）→ connection 光点
- thanks（谢谢你的分享）→ connection 光点
- hug（抱抱你）→ warmth 光点
- share（分享我的经历）→ 展开输入框，送出后双光点

每次回应：openness +3, vitality +1
分享经历：openness +5, depth +2（更高权重）

## 六、技术要点

### 水波动画

- 使用 `requestAnimationFrame` 直接更新 SVG path 的 `d` 属性
- 三层波浪，不同参数：
  - back: amp=10, freq=0.04, speed=0.7, phase=0
  - mid: amp=8, freq=0.05, speed=1.0, phase=1.5
  - front: amp=6, freq=0.045, speed=0.5, phase=3.0

### 沉淀动画

记录后：mini Bubble 液体 height 从 66% → 82%（1.5s transition）
文案每 1.8s 轮换，3s 后跳转理解页

### 漂浮粒子

- 每 2-4s 随机生成一个粒子
- 粒子大小 2-5px，随机位置
- 动画 4-10s，结束后自动移除
- 粒子数量上限 = particleDensity + 3

## 七、待完善的方向（如果继续开发）

1. **Self + Relationship + Community 三维度数据采集**：目前只采集了 Self（记录）和 Relationship（回应），Community 维度（收藏、停留、阅读）还未实现
2. **长期记忆的 AI 提取**：目前记忆是用户原文截取，未来需要 AI 提取关键线索（"被认可""自我怀疑"等主题标签）
3. **成长惊喜的触发机制**：目前成长页随时可看，未来应该改为"积累足够证据后偶尔主动提醒"
4. **影响的可视化**：目前只是一个数字，未来可以做成更温暖的可视化（比如光点传播动画）
5. **回应内容的展示**：目前用户分享的经历只是存到 localStorage，未来需要让这些回应真正"被别人看到"
6. **周期数据的真实接入**：目前周期状态是写死的"黄体期"，未来需要接入真实周期计算

## 八、产品最终气质

整个产品要让用户感受到：

> 不是 AI 在定义我。
> 而是 AI 在陪我慢慢认识自己。
>
> 不是 AI 告诉我答案。
> 而是 AI 帮我看见那些自己没有注意到的变化。
>
> Bubble 不只是记录情绪。
> Bubble 会替我记住那些我自己都会忘记的人生轨迹，并让我在与他人的真实连接中，发现自己已经悄悄成长。
