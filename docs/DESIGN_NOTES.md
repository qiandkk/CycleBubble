# CycleBubble redesign notes

## Product understanding

CycleBubble is not an emotion analysis app, a period prediction app, or a visual bubble concept. It is a cycle-based emotional understanding product.

The product helps users move from "what is wrong with me" to "this feeling may have a cycle and context background".

## Core path

Today state -> record feeling -> anonymous resonance -> pattern discovery -> understand myself.

## Product principles

- The user is the protagonist. The bubble is the living state translator and brand anchor.
- The bubble stays visually central on Today and Cycle pages, but it does not become a data dashboard.
- No diagnosis, no prediction, no emotion score, no negative emotion count.
- Keep real events and context: external evaluation, disrupted plans, fatigue, need for quiet.
- Anonymous resonance is a core feature, not an appendix.
- Review pages discover patterns instead of counting problems.

## Visual direction

Reference Apple Health, Flo, and Headspace as real product systems. Keep a breathable, alive, warm product feeling.

Use milk white, mist pink, soft lavender, and warm neutral surfaces. Avoid large deep blue and tech-blue areas. Bubble visuals should feel alive and central, not like corner badges or analytics widgets.
